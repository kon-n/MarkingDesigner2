# ワークフロー: プロジェクトドキュメントの更新

タスクを完了したり、新しい機能を追加したりした場合は、常にこのワークフローを使用してください。

1. **変更の分析**: 修正または作成したファイルを確認します。
2. **FEATURES.md の更新**:
   - 新しい機能が実装された場合は、「コアマーキング機能」または関連するセクションに追加します。
   - 完了した機能には `[x]` を、計画中の機能には `[ ]` を使用します。
3. **PROGRESS.md の更新**:
   - 完了したタスクを `[x]` でマークします。
   - 「最近のアクティビティ」セクションを現在の日付と作業内容の概要で更新します。
   - マイルストーンに到達した場合は「現在のステータス」を更新します。
4. **変更の保存**: ドキュメントの更新をコードの変更と一緒に保存してください。
